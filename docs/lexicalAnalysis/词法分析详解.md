# 词法分析目标
- 实际上，我们编写的程序文件本质上是一个txt文件，即程序文件的内容相当于一个长字符串
- 词法分析的目标就是将程序文件的内容转换为一系列的Token
  - 每个Token都有一个类型和一个值
  - 类型用于表示Token的种类，例如标识符、关键字、运算符等
  - 值用于表示Token的具体内容，例如变量名、运算符符号等
- 词法分析输出的Token将用于后续的语法分析
- **总结**
  - 输入：待分析的程序文件内容（一个长字符串）
  - 输出：Token序列（每个Token都有类型和值）
# 词法分析方法
- 首先，我们需要确定待分析的编程语言的语法构成，即关键字、标识符、运算符、分隔符等
- 然后，我们需要设计DFA（确定有限状态自动机）来实现词法分析
- 最后，我们需要实现词法分析器，将输入的程序文件转换为Token序列
